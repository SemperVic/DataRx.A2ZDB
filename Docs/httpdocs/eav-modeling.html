Entity-Attribute-Value Modeling

Today, there are several areas of concern when one discusses Integrated Definition Methods. IDEF is a group of modeling methods that can be used to describe operations in an IT enterprise. Initially abberviated as ICAM (Integrated Computer Aided Manufacturing) Definition in 1988, the ICAM definition was renamed in 1999 to IDEF. IDEF also refers to a family of modeling languages in the field of systems and software engineering. These languages cover a wide range of uses, from functional modeling to data, simulation, object-oriented analysis/design and knowledge acquisition.

A2ZDB uses a variety of IDEF methods but primary focus has been applied toward IDEF0 and IDEF1 or IDEF1X to be specific. IDEF0 is a functional modeling methodology for describing use case functionality. IDEF0 is built on the functional modeling language Structured Analysis and Design Technique

Even more focus has been placed on IDEF1X. IDEF1X is a data modeling language for the development of semantic data models. IDEF1X permits the construction of semantic data models which may serve to support the management of data as a resource, the integration of information systems, and the building of information data storage models.

IDEF helps you model data in a standard, consistent and predictable manner in order to manage it as a resource. It can be used in projects requiring a standard means of defining and analyzing the data resources within an organization. Such projects include the incorporation of a data modeling technique into a methodology, managing data as a resource, integrating information systems, or designing computer databases. The primary objectives of the IDEF1X standard are to provide:

  • Means for completely understanding and analyzing an organization's data resources
  • Common means of representing and communicating the complexity of data
  • A technique for presenting an overall view of the data required to run an enterprise
  • Means for defining an application-independent view of data which can be validated by users and transformed into a physical database design
  • A technique for deriving an integrated data definition from existing data resources.

The principal objective of IDEF1X is to support integration. The approach to integration focuses on the capture, management, and use of a single semantic definition of the data resource referred to as a "conceptual schema" in the industry and referred to as "Taxonomy Namespace" in Arizona's help documentation. The conceptual schema or taxonomy schema provides a single integrated definition of the data within an enterprise which is not biased toward any single application or tier of data and is independent of how the data is physically stored or accessed. The primary objective of this taxonomy schema is to provide a consistent definition of the meanings of and interrelationships between data that can be used to integrate, share, and manage the integrity of data. A taxonomy schema must have three important characteristics

  • Consistent with the infrastructure of the business and true across all application areas
  • Extendible, such that new data can be defined without altering previously defined data
  • Transformable to both the required user views and to a variety of data storage and access structures.
